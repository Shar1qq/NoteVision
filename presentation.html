Human-in-the-Loop

🛡️ Gate 1 — Ethical Safety Screening (Step 0)
Triggered by: exam papers · personal medical/legal records · CONFIDENTIAL-labelled docs · inappropriate content

Mechanism: Gemini screens image with structured JSON prompt before any processing begins. Pipeline halts immediately if concern detected.

User choices:
✋ Acknowledge & Take Responsibility → proceed
🚫 Cancel — Do Not Process → abort
🟢 Gate 2 — Output Approval (Final)
Always shown after all 5 pipeline steps complete.

User reviews: Refined output · Self-critique tab · Activity log · Ethics & Compliance tab

User choices:
✅ Accept Output → downloads unlocked (Markdown + DOCX)
❌ Reject Output → result discarded
🔁 Re-run Agent → fresh pipeline

Ethical Decision Making

Speed vs. Quality: A Real Dilemma

🎯 The Decision Faced
Should we deliver output immediately after conversion, or add a critique + refinement pass that costs extra time and API calls?
Option A — Speed First:
Return output immediately. Faster UX. Cheaper.
Risk: Inaccurate LaTeX, missing sections, structural errors delivered to student.
Option B — Quality First (Chosen):
Add self-critique → refinement loop. Slower but validated.
Benefit: Output is reviewed before reaching the user. Trust is maintained.
Ethical Reasoning: Academic content errors have real consequences (wrong grades, misunderstanding). Professional responsibility demanded quality over speed. This aligns with ACM §2.5 — thorough evaluation.

Agent Classification

Agent Type Selection

🎯 Goal-Based Agent
NoteVision is a goal-based agent with a fixed objective: produce the highest-quality Markdown transcription of the uploaded image.

Every decision (strategy selection, refinement trigger) is evaluated against this goal. The confidence score quantifies how well the goal was achieved.
📖 Learning Component
NoteVision has a limited learning component:
• Past conversions injected as context (memory-based adaptation)
• Critique feedback drives refinement (in-loop learning)

It does not train a model — but it adapts its output based on self-generated feedback.
Justification for Goal-Based + Semi-Autonomous: Academic content requires deterministic, auditable decision logic (rule-based strategy selection) combined with LLM intelligence for the conversion task itself — not pure machine learning, which would be opaque and unauditable.

Operational Cycle

Observe → Interpret → Decide → Act → Learn

👁️ Observe
User uploads image. Agent receives PIL Image object.
🔍 Interpret
assess_image() evaluates quality, math, diagrams, density. Returns structured ImageAssessment.
🧠 Decide
select_strategy() applies rule-based logic → picks 1 of 5 strategy prompts.
⚙️ Act
convert() sends image + strategy prompt to Gemini. Returns Markdown.
🔎 Learn (Critique)
critique() evaluates output. refine() fixes issues. Result stored in AgentMemory.
🤝 HITL Gate
User accepts/rejects/re-runs. Only accepted output can be downloaded.

Memory Architecture

Short-Term vs. Long-Term Memory

⚡ Short-Term Memory (AgentMemory.agent_short_term)
• Stores last 5 conversion results
• Injected into each new conversion as context
• Helps agent maintain consistency (e.g., same LaTeX style, same heading format)
• Cleared when window exceeds 5

Analogous to working memory in humans
📚 Long-Term Memory (AgentMemory.agent_history)
• Stores all conversions in this session
• Displayed in sidebar as "Recent History"
• Shows filename, quality, strategy, confidence
• Acts as an audit trail

Autonomy Design

Semi-Autonomous — A Justified Choice

Full Autonomy (Rejected)
Agent processes image, produces output, sends it to user automatically. No human check.
Risk: Wrong academic content delivered with no oversight.
Manual Only (Phase 1)
User does everything manually — not innovative, defeats the purpose of AI.
Semi-Autonomous (Chosen ✓)
Agent autonomously perceives, decides, converts, critiques, and refines. But human approves before any output is released.

Why justified: Academic content errors have real-world consequences. The HITL gate is a professional responsibility, not an inconvenience.
Perception: Autonomous Conversion: Autonomous Critique: Autonomous Safety Gate: HITL Final Output: HITL

Human Control

Human-in-the-Loop: Two Gates

🛡️ Gate 1 — Ethical Safety Gate (Step 0)
When triggered: Agent detects exam papers, personal records, confidential documents, or inappropriate content

What happens: Pipeline halts. User sees amber warning with concern type and recommendation.

User choices:
✋ Acknowledge & Take Responsibility → proceed
🚫 Cancel — Do Not Process → abort
🟢 Gate 2 — Output Approval Gate (Final)
Always shown after agent completes all 5 steps.

User reviews: Refined output, self-critique, activity log, ethics tab

User choices:
✅ Accept Output → downloads unlocked
❌ Reject Output → result discarded
🔁 Re-run Agent → fresh pipeline run
Key Design Principle: The agent suggests — the human decides. No output reaches the user without explicit human approval.
